## 📖 Glossário

**Termos comuns usados na documentação:**

| Termo | Significado |
|-------|-------------|
| **Candidato** | Pessoa que fez inscrição mas ainda não está matriculada |
| **Aluno** | Candidato que foi matriculado |
| **Unidade** | Campus ou filial da instituição de ensino |
| **Série** | Ano escolar (Ex: 1º ano, 2º ano, etc.) |
| **Turma** | Classe específica dentro de uma série |
| **Atendimento** | Entrevista/conversa com candidato |
| **Slug** | Identificador único usado na URL (ex: `unidade-centro`) |
| **Status** | Situação atual do candidato no processo |
| **RLS** | Row Level Security - Segurança a nível de linha do banco |
| **Migration** | Script SQL que altera estrutura do banco |
| **Profile** | Perfil/tipo de usuário (admin, direção, etc.) |
| **Supabase** | Plataforma de backend (banco + autenticação) |
| **Component** | Bloco reutilizável de interface no código |
